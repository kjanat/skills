# Rationale

Followed SKILL.md "Write new step" reading order: SKILL.md, external-files.md, examples.md, security.md. The task matches the bundled `getOctokit` cross-org example almost verbatim, so a single `actions/github-script@v9` step does both actions in one place. The default `github` client uses `GITHUB_TOKEN` for the in-repo label add; a secondary client built via the injected `getOctokit(token)` factory (not redeclared with `const`/`let` shadowing) carries the PAT for the cross-repo `createDispatchEvent`. The PAT is passed through `env` (per security rules — never inline `${{ ... }}` inside the script body), and `context.issue.number` works for both `issues.opened` and `pull_request.opened` payloads. Workflow `permissions:` are scoped to the minimum needed for the in-repo label write (`issues: write`, `pull-requests: write`); the cross-repo dispatch relies entirely on the PAT, so no extra `GITHUB_TOKEN` scope is required.
